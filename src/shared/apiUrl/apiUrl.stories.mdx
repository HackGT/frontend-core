# apiUrl

This method returns the URL for the appropriate HexLabs API service that's dependent on the
current environment. This value is determined by the `process.env.REACT_APP_API_ENVIRONMENT`
variable.

**Production**

If `process.env.REACT_APP_API_ENVIRONMENT` is set to `PRODUCTION`, the URL will be towards a
production server (like https://registration.api.hexlabs.org).

**Development**

If `process.env.REACT_APP_API_ENVIRONMENT` is set to `DEVELOPMENT`, the URL will be towards a
local development server (like http://localhost:8003).

## Import

```typescript
import { apiUrl } from "@hex-labs/core";
```

## Arguments

| Name    | Type    | Description                   |
| ------- | ------- | ----------------------------- |
| service | Service | The API service to request    |
| path    | string  | The path URL for this service |

Service can be an enum of these values:

- `Service.USERS`
- `Service.HEXATHONS`
- `Service.NOTIFICATIONS`
- `Service.REGISTRATION`
- `Service.FILES`
- `Service.AUTH`
- `Service.EXPO`
- `Service.HARDWARE`
- `Service.FINANCE`

## Return value

Returns a string value of the URL for the appropriate HexLabs service.

## Usage

### Example 1

This retrieves a url that can be used to get a list of hexathons.

```typescript
const [{ data, loading, error }] = useAxios(
  apiUrl(Service.HEXATHONS, "/hexathons")
);
```

### Example 2

Uses the file service to get the viewing link for a file id.

```tsx
<Link href={apiUrl(Service.FILES, `/files/${fileId}/view`)}>Click to View</Link>
```
